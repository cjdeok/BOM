# BOM 데이터 관리 방식 비교 분석 보고서

본 보고서는 제품별 BOM 수식 마스터 데이터를 관리하는 세 가지 주요 방식(CSV, JSON, Database)에 대한 사용자 편의성 및 서버 성능/관리 측면의 장단점을 비교합니다.

---

## 1. 관리 방식별 상세 비교

| 구분 | CSV (현재 방식) | JSON 파일 | Database (SQL) |
| :--- | :--- | :--- | :--- |
| **사용자 편의성** | **최상** (엑셀로 직접 수정 가능) | **최하** (구조적 문법 지식 필요) | **상** (전용 관리 도구/화면 필요) |
| **수정/유지보수** | 직관적이고 즉각적인 반영 | 오타 및 문법 오류에 매우 취약 | 안정적이며 이력 관리에 용이 |
| **서버 처리 속도** | 보통 (파일 I/O 발생) | 중상 (파싱 효율 높음) | **최상** (인덱싱 및 캐싱 활용) |
| **데이터 안정성** | 낮음 (파일 유실/손상 위험) | 보통 (파일 단위 관리) | **최상** (트랜잭션/백업 지원) |
| **대용량 확장성** | 낮음 (파일 수 증가 시 부하) | 보통 | **최상** (수만 건 이상 처리 가능) |

---

## 2. 주요 특징 분석

### **A. CSV (Comma-Separated Values)**
- **특징:** 표 형식의 텍스트 데이터로, 엑셀(Excel)과 완벽하게 호환됩니다.
- **장점:** 개발 지식이 없는 현업 담당자가 가장 익숙하게 데이터를 관리할 수 있습니다. 별도의 업로드 시스템 없이 파일 교체만으로 운영이 가능합니다.
- **단점:** 데이터 간의 관계 설정이 어렵고, 파일명이 틀리거나 컬럼 순서가 바뀌면 시스템 오류로 직결됩니다.

### **B. JSON (JavaScript Object Notation)**
- **특징:** 키-값(Key-Value) 쌍으로 이루어진 데이터 포맷입니다.
- **장점:** 웹 환경에서 데이터 전송 시 가장 표준화된 방식이며, BOM과 같은 계층 구조(Tree)를 표현하기에 최적화되어 있습니다.
- **단점:** 사람이 직접 읽고 쓰기에는 가독성이 떨어지며, 쉼표 하나만 빠져도 전체 데이터를 읽을 수 없게 됩니다.

### **C. DB (Database - SQLite, PostgreSQL 등)**
- **특징:** 구조화된 테이블에 데이터를 저장하고 SQL 언어로 제어합니다.
- **장점:** 검색 속도가 가장 빠르며, 수만 명의 사용자가 동시에 접속해도 안전하게 데이터를 보호합니다. 수정 이력(Log)을 남기기 좋아 보안 및 감사에 유리합니다.
- **단점:** 초기 구축 비용과 관리 시스템(Admin UI) 개발이 추가로 필요합니다.

---

## 3. 향후 전략 제안

현재 프로젝트 규모에서는 **CSV 방식의 직관성**이 업무 효율성을 높이는 데 큰 역할을 하고 있습니다. 하지만 시스템 고도화 단계에 따라 다음과 같은 로드맵을 제안합니다.

1. **1단계 (현재):** CSV 기반 운영 (엑셀을 통한 수식 자유도 확보)
2. **2단계 (중기):** CSV 업로드 기능을 통한 **DB 저장 자동화** (사용자는 엑셀로 작성하되, 서버는 DB에서 빠르게 조회)
3. **3단계 (장기):** 웹 기반 **수식 관리자 페이지** 구축 (엑셀 없이 웹에서 직접 수식 수정 및 시뮬레이션 가능)
